## 文档定位
- `项目方案`：说明为什么这么设计，适合项目负责人、初学者和接手人员。
- `完整架构版`：从系统层面讲清楚模块分工和运行流程。
- `逐函数逐行版`：面向调试和修改代码，解释关键函数和实现细节。
- `语音动作表情对照表`：方便查语音命令对应的状态、动作和表情。
- `状态机与表情联动方案`：讲清楚状态机如何驱动表情和动作。
- `天问block配置对照表`：用于对接天问/ASR 语音模块时的配置参考。
- `后期扩展操作手册`：讲如何安全扩展功能，减少改坏主工程的风险。
- `机器狗扩展方案`：给出更长远的功能扩展方向。

## 工程核心链路
工程当前的核心运行链路可以概括为：
- 语音模块通过 `USART3` 向 MCU 发送命令码。
- `asr.c` 接收命令码并转换为 `DogCommand_t`。
- `dog_state.c` 将命令映射为 `DogState_t`。
- `dog_motion.c` 根据状态选择对应动作帧和表情帧。
- `servo.c` 把动作帧转换成 5 路舵机 PWM 输出。
- `OLED` 模块负责显示对应的脸部图片和文字。

## 当前工程特点
- 使用 `STM32F103C8T6`。
- 使用 `CubeMX + Keil MDK-ARM`。
- 舵机总数为 5 个，4 个腿部舵机 + 1 个尾巴舵机。
- OLED 使用软件 I2C，硬件资源占用少，但依赖 GPIO 时序稳定。
- 状态机是轻量直映射结构，便于后续扩展。
